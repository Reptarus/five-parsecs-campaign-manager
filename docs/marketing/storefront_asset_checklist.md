# Storefront Asset Checklist

## 1. Overview

This document is a master checklist for creating and preparing all visual and text assets required for the game's store pages across all target platforms. A consistent and professional presentation is critical for attracting players.

**Target Platforms:**
-   [ ] Steam (PC/Mac/Linux)
-   [ ] Google Play Store (Android)
-   [ ] Apple App Store (iOS)

--- 

## 2. Universal Assets (Required for All Platforms)

These assets form the core of the game's brand identity.

-   [ ] **Key Art / Cover Art**: A high-resolution, compelling image that represents the game. (e.g., `4096x4096px` master file).
-   [ ] **Game Logo**: Transparent background, high-resolution. Should have variations for light and dark backgrounds.
-   [ ] **Short Description / Pitch**: A concise, exciting 1-2 sentence summary of the game.
-   [ ] **Long Description**: A detailed description covering core features, the story premise, and what makes the game unique. Should be formatted with Markdown/BBCode for readability.
-   [ ] **Gameplay Trailer**: A 1-2 minute trailer showcasing core gameplay loops (character creation, combat, campaign map, events). Should end with a clear call to action and logo.
-   [ ] **Screenshots (Core Set)**: A set of 10-15 high-quality, compelling screenshots showing diverse aspects of the game. **AVOID** just taking pictures of menus.
    -   [ ] Character creation in progress.
    -   [ ] An intense combat moment.
    -   [ ] The campaign map view.
    -   [ ] A narrative event dialog.
    -   [ ] A psionic power being used.
    -   [ ] A view of the crew/roster screen.

## 3. Steam (PC/Mac/Linux)

-   **Reference**: [Steam Graphical Asset Rules](https://partner.steamgames.com/doc/store/assets)

| Asset Type                | Required Size(s)        | Status      | Notes                                                                 |
| ------------------------- | ----------------------- | ----------- | --------------------------------------------------------------------- |
| **Header Capsule**        | `460x215px`             | [ ] Done    | The most common banner for the game on Steam.                         |
| **Small Capsule**         | `231x87px`              | [ ] Done    | Used in various "more like this" and list views.                      |
| **Main Capsule**          | `616x353px`             | [ ] Done    | A larger, more prominent banner.                                      |
| **Vertical Capsule**      | `374x448px`             | [ ] Done    | Used for the new library view.                                        |
| **Page Background**       | `1438x810px` (Optional) | [ ] Done    | A subtle background image for the store page.                         |
| **Screenshots**           | `1280x720` or `1920x1080` | [ ] Done    | Minimum of 5.                                                         |
| **Library Capsule**       | `600x900`               | [ ] Done    | The main "box art" for the new Steam library.                         |
| **Library Hero**          | `1920x620`              | [ ] Done    | The banner image at the top of the library page.                      |
| **Library Logo**          | `640x360` (Transparent) | [ ] Done    | A transparent logo for placement over the Library Hero image.         |

## 4. Google Play Store (Android)

-   **Reference**: [Google Play Graphical Asset Guidelines](https://support.google.com/googleplay/android-developer/answer/9866151)

| Asset Type                | Required Size(s)        | Status      | Notes                                                                 |
| ------------------------- | ----------------------- | ----------- | --------------------------------------------------------------------- |
| **App Icon**              | `512x512px` (32-bit PNG) | [ ] Done    | Must be a perfect square.                                             |
| **Feature Graphic**       | `1024x500px`            | [ ] Done    | The main banner at the top of the store listing. Can include video link. |
| **Screenshots**           | Min `320px`, Max `3840px` | [ ] Done    | 2 to 8 screenshots for phones. Can also add tablet-specific shots.    |
| **Promo Video**           | YouTube URL             | [ ] Done    | Optional, but highly recommended.                                     |

--- 

## 5. Apple App Store (iOS)

-   **Reference**: [App Store Screenshot Specifications](https://developer.apple.com/help/app-store-connect/reference/screenshot-specifications/)

| Asset Type                | Required Size(s)                               | Status      | Notes                                                                 |
| ------------------------- | ---------------------------------------------- | ----------- | --------------------------------------------------------------------- |
| **App Icon**              | Multiple sizes required, managed by Xcode.     | [ ] Done    | Create a `1024x1024px` master file.                                   |
| **Screenshots**           | `1242x2688px` (6.5"), `1242x2208px` (5.5") etc. | [ ] Done    | Must provide for the largest phone and tablet sizes. Can be scaled down. |
| **App Previews**          | Up to 3 short videos (15-30 seconds).          | [ ] Done    | The App Store's version of a trailer.                                 |
| **Promotional Text**      | 170 characters                                 | [ ] Done    | Appears above the description. Can be updated anytime.                |

--- 

## 6. DLC Content

For the **Compendium DLC**, a smaller, focused set of assets will be needed for its own store page on platforms that support it (like Steam).

-   [ ] **DLC Header Capsule (Steam)**: `460x215px` that clearly indicates it's an expansion.
-   [ ] **DLC Description**: Text that clearly explains what new features are included in the expansion.
-   [ ] **DLC Screenshots**: 5-10 screenshots specifically showcasing DLC content (Psionics, new species, new missions).
